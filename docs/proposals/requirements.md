# Initial Requirements

## 背景と課題

- Slackの分報文化は有用だが、チームの目があるため書く内容を無意識にフィルタリングしてしまう
- 既存のSNS（X, Threads等）は「他者の反応」を前提とした設計で、純粋な記録に向かない
- 仕事中にサッと書けて、自分だけが見られる分報のような場所が欲しい

## コンセプト

**自分専用の非公開型マイクロブログ（パーソナル分報）**

- SNS要素（フォロー、いいね、リポスト、拡散）を一切排除
- 手軽に投稿できる（XのようなUI）
- デスクトップの隅に小さなウィンドウで常時表示して使う
- 自分だけが閲覧・投稿できる完全プライベート空間

## 主要機能

- テキスト投稿（短文、マイクロブログ形式）
- タグによる分類・検索
- タイムライン表示（時系列）
- OAuth認証（GitHub / Google）による自分専用アクセス

### 使用イメージ

1. ログインする（OAuth）
2. テキストを入力し、必要に応じてタグを付ける
3. 投稿ボタンを押す
4. タイムラインに自分の投稿が時系列で並ぶ
5. タグや検索で過去の投稿を振り返る

## プラットフォーム

- Webアプリケーション（デスクトップ利用が前提。小さいウィンドウでの常時表示に最適化）

## 技術スタック

- フレームワーク: TanStack Start (フルスタックReactフレームワーク)
- ルーティング: TanStack Router (型安全)
- バックエンド: Supabase (認証、DB、ストレージ)
- ビルドツール: Vite
- UIライブラリ: 未定（セットアップ時に決定）

## 非機能要件

- 完全非公開: 認証済みユーザー（自分）のみアクセス可能
- コンパクトUI: 360px幅のウィンドウでもレイアウトが崩れない
- 高速な投稿体験: 入力から投稿完了まで最小限のステップ（5秒以内）
- データの永続性: 投稿データの安全な保存
