# チェックポイント

引数: `/checkpoint [任意のメモ]`

## 手順

1. 現在の作業状態を確認:
   - `git status` で変更ファイルを確認
   - 対象の `.steering/*/tasklist.md` を読み込み、進捗を確認

2. 状態サマリーを作成:

```
## Checkpoint: [日時]

### 完了した作業
- [完了タスクのリスト]

### 進行中の作業
- [現在取り組んでいるタスク]

### 未着手の作業
- [残りのタスク]

### 変更ファイル
- [git statusの結果]

### メモ
$ARGUMENTS
```

3. サマリーを `.steering/` 内のアクティブなディレクトリに永続化:
   - 保存先: `.steering/[日付]-[機能名]/checkpoint.md`
   - 既にファイルが存在する場合は上書きする（最新状態のみ保持）

4. サマリーをユーザーに表示
5. コンパクション前の状態保存として機能する。新しいセッションでは `checkpoint.md` を読み込むことで作業を再開できる
